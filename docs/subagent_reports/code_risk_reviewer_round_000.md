# code_risk_reviewer report

## Blockers

- `scripts/slurm_run_preexp.sh:34,45-48`, `scripts/slurm_run_smoke.sh:19-25,36-42`, `scripts/slurm_run_eval_all.sh:24-26`: still call `sbatch` directly and use `--export=ALL`; this bypasses `goal_submit_batch.py` and can export tokens/credentials into jobs; disable these launchers, rename them as deprecated non-executable docs, or make them delegate to `scripts/goal_submit_batch.py` with `--export=NONE`.
- `scripts/run_preexp.sh:8-14`, `scripts/run_smoke.sh:8-14`, `src/run_all.py:62-76`: can run training directly on the login node; this violates the project protocol and can start expensive local work accidentally; add a hard guard requiring `SLURM_JOB_ID`/explicit small local flag, or remove these as autonomous entry points.

## High severity

- `scripts/goal_submit_batch.py:275-287`, `scripts/goal_validate_manifest.py:111-129`: `max_concurrent_gpu_jobs` is validated as a number but not enforced against experiment count or submission fan-out; a manifest can submit many train jobs at once; reject manifests where active train jobs exceed the limit, or implement dependency/throttling.
- `scripts/goal_submit_batch.py:70-74,173-174`: `SBATCH_ARGS` is accepted via `shlex.split` without an allowlist; users can inject Slurm semantics such as `--export`, `--wrap`, `--array`, or conflicting dependencies; allow only scheduler/site options like account/partition/qos/time and reject export/wrap/array/dependency.
- `scripts/goal_validate_manifest.py:106-109,168-171`, `scripts/goal_submit_batch.py:126-139`: output roots are only blocked under `outputs/baselines`, but absolute paths and `../` paths are otherwise allowed; this permits path traversal and unintended writes outside the goal output tree; require relative paths under `outputs/goal/runs` and `outputs/goal/eval`.
- `scripts/goal_validate_manifest.py:41-45`, `scripts/goal_init.py:21-31`: frozen baseline validation only checks that two files exist; stale, mismatched, wrong-task, or tampered baseline files can mark the repo ready; load `baseline_manifest.json`, validate metric/tasks/hash, and verify the summary matches the manifest.
- `scripts/goal_collect.py:62-64,105-127`, `scripts/goal_scoreboard.py:171-174`: candidate IDs are derived from observed loop outputs, not manifest-declared final candidates; final runs with `eval_all_loops=true` can create post-hoc loop-depth candidates after seeing final results; require final manifests to predeclare candidate IDs/loop indices and score only those.

## Medium severity

- `scripts/goal_validate_manifest.py:173-176`, `scripts/goal_submit_batch.py:95-102`: `purpose: final` can still use per-experiment `eval.task_names` subsets; scoreboard will later fail missing tasks, but validation should reject this before submission; require exact protocol task coverage for every final experiment.
- `src/eval_mteb.py:46-64,217-225`: metric parsing recursively accepts the first matching alias anywhere in raw MTEB output; if MTEB returns multiple splits or nested metrics, `ndcg_at_10` can be ambiguous; parse the intended retrieval split/schema explicitly and fail on multiple matches.
- `scripts/goal_common.py:151-158`, `scripts/goal_collect.py:182`, `scripts/goal_scoreboard.py:262`: CSV writes are non-atomic while JSON state writes are atomic; interrupted or concurrent collection/scoring can leave truncated CSVs; write CSVs through temp files plus `os.replace`, optionally with a lock.
- `scripts/goal_collect.py:160-162`: `partial_tasks` status is overwritten to `invalid_metric` when any invalid row exists; this loses the more specific failure class; preserve all reasons or keep `partial_tasks` when task coverage is incomplete.
- `scripts/goal_preflight.py:54-56`: preflight mutates `outputs/goal` via dry-run submission and only self-tests scoreboard; repeated preflight is not purely read-only and does not test manifest rejection, baseline validation, collection, path traversal, or NaN/duplicate handling; add isolated tempdir self-tests for each guardrail.

## Low severity

- `scripts/*.sh`, `scripts/*.sbatch`: shell entry points have no `--help`/usage output; users may run legacy scripts by mistake; add a usage guard or deprecation message for unsafe legacy launchers.
- `scripts/__pycache__/*`: generated bytecode exists locally under `scripts`; it is ignored, but should not be treated as source infrastructure; clean locally before packaging/review if needed.
- `scripts/slurm_train.sbatch:24-26`, `scripts/slurm_eval.sbatch:24-26`: logs print conda env and Python path; not credentials, but can expose local machine paths in Slurm logs; keep logs ignored and avoid committing them.

## Positive checks

- `scripts/goal_submit_batch.py:37-45,251-265`: defaults to dry-run, requires explicit `--submit`, and refuses real non-smoke submit without a frozen baseline.
- `scripts/goal_submit_batch.py:23-34,57-67`: new submission path uses `--export=NONE,...` with a small runtime whitelist, avoiding broad environment dumps.
- `scripts/goal_common.py:184-197`, `scripts/goal_scoreboard.py:119-148`: NaN/inf/non-numeric metrics do not score as wins, and `pass_all_tasks` requires every final task to be valid and above margin.
- `scripts/goal_validate_manifest.py:137-151`: duplicate and unsafe `run_id` values are rejected.
- `scripts/goal_freeze_baseline.py:76-83`: frozen baseline overwrite is refused unless `--force` is explicit.
- `src/eval_mteb.py:151-184`: evaluation summary append has a lock and dedupes `(version, task, loop_idx, checkpoint_dir)`.
- No unsafe `rm -rf`, broad deletes, `printenv`, `.env` reads, token cache reads, `scancel`, or Slurm-modifying commands were found in reviewed diff/scripts.

## Required fixes before preflight

- Disable or route all direct `sbatch` launchers through `scripts/goal_submit_batch.py`.
- Guard or remove login-node training entry points.
- Enforce manifest output roots under approved `outputs/goal/**` directories.
- Validate frozen baseline manifest, task coverage, metric, and hash before state or submission readiness.
- Enforce `max_concurrent_gpu_jobs` in validation/submission.
- Sanitize `SBATCH_ARGS` with a strict allowlist.
- Require predeclared final candidate IDs/loop indices and exact final-task coverage.
- Add cheap self-tests for manifest rejection, baseline validation, collection failures, NaN/duplicate rows, and path traversal.

## Uncertainty

- I did not run preflight/tests or any Slurm commands, per read-only/no-submit instructions.
- I did not inspect ignored `outputs/**` contents or any credential-like files.
- Some legacy scripts may be intended as deprecated, but they are still executable and not guarded, so I treated them as active risks.
